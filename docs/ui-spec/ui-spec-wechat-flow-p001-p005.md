### P-001: 编辑器主页（`/` 或 `/docs/:docId`）

**映射功能**: F-001, F-002, F-003, F-004, F-011, F-012, F-014

**使用组件**: C-001（顶栏）, C-002（Splitter）, C-003（工具栏按钮）, C-004（SourcePane）, C-005（PreviewPane）, C-006（左侧面板 Tab）, C-007（主题卡片）, C-008（Block 库列表项）, C-009（命令面板）, C-013（诊断面板）, C-017（ZhTypoReviseDialog）, C-018（ImageUploadOverlay）, C-019（PaintDrawer）, C-020（BaseColorDeriveModal）, C-021（DirectiveAutocompletePopover）

```

└─────────────────────────────────────────────────────────────────────────────┘
```

视觉重心：编辑区（最亮暖白 `#FAF8F5`）是视觉重心，左栏（略深 `#F4F1EC`）与预览区（微冷 `#F7F7F7`）通过背景色梯度形成层次。三栏之间由 `4px` Splitter 连接，Splitter 颜色 `--color-border-subtle` 保证存在感低。

关键间距：
- 左栏内边距：`var(--space-4)`（左右各 16px）
- 编辑器输入区 padding：`var(--space-6) var(--space-8)`（上下 24px，左右 32px）
- 预览面板内边距：`var(--space-4)`（周围 16px 留白，iframe 居中）

**状态流**：

| 状态 | 视觉表现 |
|------|---------|
| `loading`（应用初始化中）| 三栏背景色铺满，C-004 区域展示骨架屏（3 条宽度各异的灰色矩形，pulse 动画），C-005 区域展示旋转 spinner（居中 `--color-brand-muted`），左栏 Tab 下方显示 3 条宽度随机的灰色占位块 |
| `empty`（无文档，首次访问）| 编辑区居中展示空状态引导，自动跳转到 P-003 主题模板市场（路由 `/themes`）让用户选择写作起点；跳转前短暂展示过渡动画（300ms）；路由跳转 `/` → `/themes` 使用 `history.replace`（非 push），用户在 P-003 按浏览器后退键时直接离开应用，避免后退键回到空状态死循环；若用户取消主题市场直接返回，则 `/` 路由保持编辑器空状态，展示「从主题模板市场新建」文字链接（`--color-brand`）；预览区显示「预览将在这里展示」占位文字 |
| `populated`（有内容）| 正常三栏显示，所有组件激活状态 |
| `error`（初始化失败，如 IndexedDB 不可用）| 全屏错误页：居中图标（`--color-error`, 48px）+ 错误说明文字 + 「刷新页面」按钮；不显示三栏 |
| `focus-mode`（F11 专注）| 左栏和右栏隐藏（display: none，不做动画以保持即时感），顶栏收缩（工具栏按钮组隐藏），编辑区铺满页面宽度，底部状态栏保留 |

### P-002: 文档列表（侧栏内嵌 / 移动底部抽屉）

**映射功能**: F-001 (AC-005 多文档管理)；对应 ARCH M-013 IndexedDB Document

**使用组件**: C-006 Tab 3（桌面/平板），C-010（操作菜单），C-011（Toast）

#### 桌面/平板：左侧面板「文档」Tab 内嵌

**空间构成**: Tab 内容区上部为操作区（高 `40px`，含「+ 新建」全宽按钮），下部为可滚动列表（剩余高度）。视觉重心在列表区，操作区紧凑不分散注意。列表项之间用 `1px solid --color-border-subtle` 细分隔，整体节奏紧凑（每项 `44px` 高 + 内部 `var(--space-3)` padding），保证窄左栏（200~320px）也能显示 ≥10 项。

**布局**：占据左侧面板 Tab 3 的内容区，顶部有「+ 新建」按钮（全宽，点击跳转到 P-003 主题模板市场选写作起点，不直接创建空文档），下方文档列表（可滚动）。

每条文档列表项（高 `44px`）：
- 文档标题（14px, `--color-text-primary`，overflow ellipsis）
- 副标题：最后修改时间（11px, `--color-text-muted`）
- 右侧三点菜单（hover 时出现，触发 C-010）

**状态流**：

| 状态 | 视觉表现 |
|------|---------|
| `loading` | 列表区域展示 3 条 skeleton 行（宽度 70% / 50% 交替），pulse 动画 |
| `empty` | 居中展示「还没有文档」（14px, muted）+ 「创建第一篇」文字链接（`--color-brand`） |
| `populated` | 文档列表正常显示，当前打开文档项背景 `--color-brand-subtle`，文字 `--color-brand`，左侧 `2px solid --color-brand` 指示条 |

**当前文档激活指示**：列表项左侧竖条 `2px solid --color-brand`，背景 `--color-brand-subtle`，文字 `--color-brand`。

#### 移动端：底部抽屉（Bottom Sheet）

**触发**：底部固定栏「文档切换」按钮（见 P-005）。

**布局**：从底部向上滑入（translateY 100% → 0，`--duration-base`），高度自适应（最大 `60vh`），顶部圆角 `--radius-lg`，顶部有拖拽指示条（宽 `40px`，高 `4px`，圆角 `--radius-full`，颜色 `--color-border`）。

**状态流**：

| 状态 | 视觉表现 |
|------|---------|
| `loading` | 3 条 skeleton 行（同桌面态） |
| `empty` | 居中「还没有文档」+ 文字链接（移动端提示需在桌面端创建） |
| `populated` | 文档列表正常显示，当前打开文档有对勾标记 |

### P-003: 主题模板市场（`/themes`）

**映射功能**: F-003 (AC-001 主题选择, AC-012 内置 template), F-008 (主题预设变体)；对应 ARCH M-005

**使用组件**: C-003（工具栏按钮）, C-022（TemplateThemeCard）, C-011（Toast）, C-014（安装进度）

**空间构成**: 三段式纵向流 — 顶部标题区（`80px`，重视觉权重 + 留白引导）、中部筛选栏（`48px`，操作区域）、底部卡片网格（剩余高度，可滚动）。视觉重心在卡片网格，标题区通过 `var(--space-8)` 大留白引出选择仪式感；筛选栏与卡片网格之间无明显分隔（依靠间距 `var(--space-6)` 自然过渡）。卡片之间 gap `var(--space-6)` 保证视觉呼吸。

每张卡片为 (主题, template) 组合：一个主题下有 ≥1 张 template 卡；5 个内置主题各至少提供 1 张 template，基础卡片数 ≥5，可扩展更多 template。卡片内容（缩略图 + 预设 Markdown + 主题配色）同时承担"写作起点"与"主题能力活样本"双重职责，兼具白名单覆盖（9 基础 Markdown 元素 + ≥6 核心 Block 容器）。

**布局**（桌面档 ≥1280px）：

```
┌────────────────────────────────────────────────────────────────┐
│ C-001 TopBar（简化版，无主题指示器）[← 返回编辑器]              │
├────────────────────────────────────────────────────────────────┤
│ 页面标题区（高 80px，padding --space-8）                        │
│  「主题模板市场」(24px, bold) + 副标题文字 (14px, muted)        │
├─────────────────────────────────────────────────────────────────┤
│ 筛选栏（高 48px）:                                              │
│  [全部] [简约通用] [生活杂志] [文艺人文] [商业财经] [科技数码]   │
│  | 搜索框（宽 240px）                                          │
├─────────────────────────────────────────────────────────────────┤
│ TemplateThemeCard 网格（4 列，gap --space-6，padding --space-8）│
│  每卡：(主题名, template 名) 组合；缩略图为实际渲染预览         │
│  卡片下方双行标题：主题名 / template 名                         │
│  「使用此模板」按钮（C-003 primary）                            │
└─────────────────────────────────────────────────────────────────┘
```

卡片（C-022 TemplateThemeCard）尺寸：宽 `280px`，高 auto；内含缩略图（高 `160px`，`--radius-sm`）+ 主题名（16px, semibold）+ template 名（14px, muted）+ 简短描述（12px, muted）+「使用此模板」按钮（C-003 primary）。

**响应式策略**（与主卷 §5 三档对齐）:

| 档位 | 网格列数 | 关键差异 |
|------|---------|---------|
| 桌面（≥1280px）| 4 列 | 标准布局；筛选栏右对齐搜索框 |
| 平板（768-1279px）| 2 列 | 卡片宽度自适应放大；筛选栏 Tab 与搜索框上下两行排列（搜索框全宽）|
| 移动（<768px）| 1 列列表（非卡片）| 每项水平排列：左侧 `64×64px` 缩略图 + 右侧主题名 + template 名 + 「使用」按钮（C-003 primary，紧凑尺寸）；顶部页面标题区高度收缩至 `56px`，副标题省略 |

平板和移动档的卡片网格 gap 同步降级：桌面 `var(--space-6)` → 平板 `var(--space-4)` → 移动 `var(--space-3)`。

**「使用此模板」按钮行为**：复制该 template 的预填 Markdown 内容到新文档 + 设置 frontmatter `theme: <themeId>` + `template: <templateId>` → 跳转 P-001 编辑器。

**状态流**：

| 状态 | 视觉表现 |
|------|---------|
| `loading-templates`（加载模板列表）| 网格区展示 8 张 skeleton 卡片（灰色矩形，pulse 动画，尺寸与真实卡片一致），移动档 skeleton 为 3 行单列 |
| `browsing`（正常浏览）| 所有 TemplateThemeCard 正常显示，当前已应用主题对应卡片有「正在使用」徽章（`--color-brand-subtle` 背景）|
| `creating-doc`（正在从模板创建文档）| 被点击卡片上显示 spinner（遮罩 0.5 opacity），C-014 Job 进度条在卡片底部展开（创建文档并复制 template 内容） |
| `document-created`（文档创建成功）| Toast（success）提示「已从模板新建文档」；1s 后自动跳转 P-001 |
| `error`（创建失败）| Toast（error）提示失败原因；卡片恢复可操作状态 |

### P-004: 设置 / 偏好（`/settings`）

**映射功能**: F-009 (主题继承与品牌包, AC-002 品牌包锁定), F-001 (AC-007 输入辅助开关), F-012 (同步配置)

**使用组件**: C-001（顶栏）, C-003（工具栏按钮）, C-012（Modal）, C-011（Toast）

**空间构成**: 经典「双栏 master-detail」结构 — 左侧导航栏（`200px` 宽，背景 `--color-surface-elevated`，纵向选项列表）+ 右侧设置内容区（最大宽 `640px`，居中，背景 `--color-surface`）。视觉重心在内容区，左侧导航通过略深背景与较弱视觉权重让位于内容。内容区两侧 `padding: var(--space-8)` 留白引导阅读节奏；分组之间 `var(--space-6)` 间距，每个表单字段之间 `var(--space-4)`。

**响应式策略**: 平板（768-1279px）双栏布局不变，左侧导航栏可折叠为 ≤120px 仅显示图标；移动（<768px）导航栏收为顶部分类标签栏（水平横向滚动，标签 chips 形式），内容区铺满。

**布局**（桌面档 ≥1280px）：

```
┌──────────────────────────────────────────────────────────────┐
│ C-001 TopBar（简化版）[← 返回编辑器]                          │
├────────────────────┬─────────────────────────────────────────┤
│                    │                                         │
│ 左侧导航栏         │ 设置内容区                              │
│ 宽 200px           │ 最大宽 640px，居中                      │
│                    │                                         │
│ - 编辑器           │ （对应当前选中的设置分组）               │
│ - 主题与品牌       │                                         │
│ - 同步与协作       │                                         │
│ - 图床配置         │                                         │
│ - API 密钥         │                                         │
│ - 关于             │                                         │
│                    │                                         │
└────────────────────┴─────────────────────────────────────────┘
```

各设置分组（内容区）：

**编辑器**：
- 自动加空格（中英文之间）— 开关（默认关；与 F-014 批量修订正交，对应 PRD F-001 AC-007 可选实时输入辅助）
- 智能引号转换 — 开关（默认关；同上）
- 字体大小（滑块，`13px ~ 20px`）
- 行高（下拉，紧凑/标准/宽松）

**主题与品牌**：
- 当前品牌包（显示名称 + 「解除锁定」文字链接，若无则显示「未配置」+ 「配置品牌包」按钮）
- 品牌包锁定的 Token 预览（色块列表 + 字体名）

**同步与协作（F-012）** `[v1 不交付 / F-012 整族 ARCH 预留]`：
> v1 该分组隐藏，仅在 ARCH 预留拓扑被启用时可见。以下为 v2 视觉规格准备，v1 不渲染。
- 实时协作开关（默认关，开启时提示 WebSocket 服务配置）
- 同步服务地址输入框
- 当前同步状态显示

**API 密钥**：
- 微信公众号 AppID/AppSecret 配置（密码框，不明文显示，hover 显示「眼睛」图标按钮切换明/暗）

**图床配置**（F-006 AC-001 支持 6 种图床：本地 / 七牛 / 阿里云 OSS / 腾讯云 COS / SM.MS / 自定义）：

每种图床显示为一行可折叠卡片（高 `48px` 折叠态，展开后高度自适应）：

```
┌─────────────────────────────────────────────────────────┐
│ [图床图标 24px] 七牛云 (Qiniu)        [启用 ⬤]  [展开 ⌄] │ <- 折叠态行（高 48px）
└─────────────────────────────────────────────────────────┘

展开后:
┌─────────────────────────────────────────────────────────┐
│ [图床图标 24px] 七牛云 (Qiniu)        [启用 ⬤]  [收起 ⌃] │
│ ─────────────────────────────────────────────────────── │
│ AccessKey:  [████████████████████████]  [👁]            │ <- 密码框 + 显隐
│ SecretKey:  [████████████████████████]  [👁]            │
│ Bucket:     [my-bucket                ]                 │ <- 普通输入框
│ Domain:     [https://cdn.example.com  ]                 │
│ 区域:       [华东 ▾]                                    │ <- 下拉选择
│                                                         │
│ [测试连接]  [保存]  [清除配置 (destructive)]              │
│                                                         │

```

每行卡片字段集合 `[ASSUMPTION]`：

| 图床类型 | 配置字段 |
|---------|---------|
| 本地（无需配置）| 仅启用开关 + 本地存储路径展示 |
| 七牛云 | AccessKey / SecretKey / Bucket / Domain / 区域（下拉） |
| 阿里云 OSS | AccessKey / SecretKey / Bucket / Endpoint / Region |
| 腾讯云 COS | SecretId / SecretKey / Bucket / Region |
| SM.MS | Token（单字段）|
| 自定义 | API Endpoint / Auth Header（key+value 表单）/ Upload Field Name / Response JSON Path |

测试连接按钮调用 backend `POST /api/v1/images/upload` 上传 `1×1px` 透明 PNG 验证；测试结果以 Toast（success/error）反馈 + 卡片底部状态行更新。具体字段集合在 dev-plan 阶段与各图床 SDK 接入工程师对齐后确定，本节字段为常见配置项推断。

**状态流**：

| 状态 | 视觉表现 |
|------|---------|
| `loading` | 设置内容区展示 skeleton（每行高 `32px`，宽度 70%，pulse 动画） |
| `empty` | 非预期状态，一般设置页总有默认值，不出现真正的空状态 |
| `populated` | 正常显示各设置项及当前值 |
| `saving` | 保存按钮变为 loading 态（spinner），成功后 Toast（success）「设置已保存」 |
| `error`（保存失败）| Toast（error）「保存失败，请重试」；各设置项恢复可编辑状态 |

### P-005: 移动端只读预览（`/preview/:docId`）

**映射功能**: F-004 (AC-001 一键复制), F-001 (AC-005 多文档切换)；与 ARCH §5.4 兼容性矩阵对齐（vw < 768px 不支持编辑）

底部固定栏细节：
- **文档切换器**（左侧）：宽 `160px`，样式为带下拉箭头的 select 样式组件，展示当前文档名，点击触发 P-002 底部抽屉（文档列表）

**状态流**：

| 状态 | 视觉表现 |
|------|---------|
| `loading` | PreviewPane 区域展示居中 spinner（`--color-brand-muted`，32px），底部固定栏正常显示但复制按钮 disabled |
