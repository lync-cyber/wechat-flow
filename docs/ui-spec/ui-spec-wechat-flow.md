## 0. 设计方向

### 0.1 产品调性

wechat-flow 是面向微信公众号写作者的专业写作与排版工具。目标用户是长期与文字打交道的内容创作者：文章质量是其核心诉求，工具应让写作者「不被 UI 打扰」。设计调性定义为：

**专注写作 — 暖意文人美学**

- 底色采用暖白 `#FAF8F5`（宣纸感），让长时间盯屏的眼睛感受温和而非刺眼的纯白
- 主题色采用墨绿 `#2D5A4E`，来自中国传统水墨的深绿，沉静内敛，适合长文写作语境
- 正文排版使用衬线字体，呼应纸质书阅读质感，降低长文阅读的认知负担
- UI 控件（按钮、标签、输入框）使用无衬线字体，保持功能性清晰
- 视觉层次通过色度差与背景色深浅变化建立，避免过多色彩抢走写作者注意力
- 装饰性元素保持克制：边框细而淡，圆角小，不使用重色阴影

### 0.2 核心原则

1. **内容优先**：编辑区（CodeMirror 6 源码面板）和预览区是视觉重心，左侧面板与顶栏服务于工具访问，不主导视野
2. **即见即得**：预览实时刷新（≤300ms debounce），写作者无需手动触发，修改立即可见
3. **层次分明**：三栏布局通过背景色梯度区分职能——左栏功能面板用略深暖灰，中栏编辑区用最亮暖白，右栏预览区用略冷背景模拟微信阅读环境
4. **专注模式**：F11 全屏专注，隐藏一切非写作元素，仅保留编辑区
5. **可感知状态**：每个交互元素必须有视觉可辨的状态变化，避免让写作者猜测工具是否在工作

### 0.3 视觉语言摘要

| 维度 | 决策 | 理由 |
|------|------|------|
| 主底色 | 暖白 `#FAF8F5` | 降低长时间阅读的视觉疲劳，贴近纸质书质感 |
| 主题色 | 墨绿 `#2D5A4E` | 稳重内敛，适合工具类界面，与暖白形成高对比 |
| 强调色 | 赤陶 `#B94A3E` | 暖色系警示与高亮，与暖白调性一致，避免冷色蓝 |
| 正文字体 | 衬线体栈（LXGW WenKai / Source Han Serif SC + Georgia） | 公众号写作内容即为正文，衬线体降低阅读负担 |
| UI 字体 | 无衬线体栈（Inter / system-ui + 思源黑体） | 功能按钮、标签、菜单需要清晰紧凑的字形 |
| 圆角 | 小圆角 `4px`（控件）/ `2px`（输入框）/ `0`（全宽分隔线） | 内容工具需要精准感，不适合圆润的消费品风格 |
| 阴影 | 极浅暖调投影，仅用于浮层（下拉菜单、Modal） | 避免过重阴影破坏纸质质感 |

### 0.4 决策记录

设计阶段共 6 个由用户拍板的关键决策点，按顺序记录如下，便于下游团队溯源：

| 决策号 | 问题 | 选项 | 决策 | 依据 |
|--------|------|------|------|------|
| Q1 | 设计调性方向（影响主色系 / 字体 / 整体色调） | A 专注写作（暖白+墨绿+衬线）/ B 现代编辑器（白+靛蓝+无衬线）/ C 专业工具深色 | **A 专注写作** — 暖白 `#FAF8F5` + 墨绿 `#2D5A4E` + 衬线中文正文 | 用户决策；目标用户为内容创作者，"专注写作"差异化定位优先于通用编辑器对标 |
| Q2 | 主布局拓扑 | A 三栏固定 / B 可调比例三栏+专注模式 / C 二栏+抽屉 | **B 可调比例三栏 + 专注模式** — 默认 200/auto/320，F11 全屏专注 | 用户决策；同时兼顾宽屏写作体验与窄屏适应 |
| Q3 | 主题切换入口位置 | A 顶栏下拉 / B 左侧面板首位 Tab / C 命令面板 | **B 左侧面板首位 Tab** — Tab 序列：主题 / 组件 / 文档；顶栏仅保留当前主题状态指示 | 用户决策；主题切换为中频操作，左栏整合 Theme + Block 收纳合理 |
| Q4 | 预览刷新策略可见性 | A 隐式即时 / B 节流+进度指示器 / C 显式按钮+自动兜底 | **A 隐式即时刷新** — `≤300ms` debounce，无刷新按钮、无进度指示器 | 用户决策；与"内容优先"设计原则一致，减少 UI 干扰 |
| Q5 | 响应式断点策略 | A 仅桌面 ≥1280px / B 平板 ≥1024px 二栏 / C 移动 ≥375px 只读 | **桌面 + 平板 + 移动三档**（用户细化方案） — 桌面 ≥1280 三栏；平板 768-1279 二栏+左抽屉；移动 <768 单栏只读 + 底部固定栏（文档切换 + 一键复制） | 用户决策；细化方案比原选项更精确 |
| Q6 | 命令面板 Ctrl+K 覆盖范围 | A 仅 UI 动作 / B UI+内容编辑 / C 全集含 LLM/MCP | **A 仅 UI 动作** — 视图/主题/导出/帮助 4 组，~15-20 命令 | 用户决策；v1 控制命令面板复杂度，B/C 留待后续迭代 |

决策依据均可在 `docs/EVENT-LOG.jsonl` 的 `user_decision` 事件中追溯（事件 `phase=ui_design`，`detail` 字段含完整 Q→A 映射）。

## 1. 设计系统

### 1.1 色彩 Token

色彩从暖白 `#FAF8F5` 基底出发，派生暖灰阶；功能色从暖色调选取，避免高饱和的 Bootstrap 原色。

#### 1.1.1 表面色（Surface）

| Token | 值 | 用途 |
|-------|----|------|
| `--color-surface` | `#FAF8F5` | 主底色、编辑器背景 |
| `--color-surface-elevated` | `#F4F1EC` | 左侧面板背景（比编辑器略深，暗示功能区） |
| `--color-surface-overlay` | `#EDEAE3` | 悬停项背景、激活菜单项 |
| `--color-surface-sunken` | `#E7E3DA` | 输入框背景（低于周围层级） |
| `--color-surface-preview` | `#F7F7F7` | 右侧预览面板底色（模拟微信默认白底，略冷以区分编辑区） |

#### 1.1.2 边框色（Border）

| Token | 值 | 用途 |
|-------|----|------|
| `--color-border` | `#D9D4CB` | 标准边框（输入框、卡片轮廓） |
| `--color-border-strong` | `#C2BCB1` | 强调边框（分隔线、选中项轮廓） |
| `--color-border-subtle` | `#E8E4DC` | 弱化边框（仅用于区域划分，不希望边框喧宾夺主） |

#### 1.1.3 文字色（Text）

| Token | 值 | 用途 |
|-------|----|------|
| `--color-text-primary` | `#1C1917` | 主正文，源码编辑器文字 |
| `--color-text-secondary` | `#4A4541` | 次要信息、标签文字、说明文字 |
| `--color-text-muted` | `#7A746C` | 辅助性文字（占位符、时间戳、禁用状态下的文字） |
| `--color-text-inverse` | `#FAF8F5` | 深色背景上的文字（主题色按钮内文字） |
| `--color-text-link` | `#2D5A4E` | 链接色，与主题色一致 |
| `--color-text-link-hover` | `#1E3D35` | 链接悬停色（主题色加深 20%） |

#### 1.1.4 主题色（Brand）

| Token | 值 | 用途 |
|-------|----|------|
| `--color-brand` | `#2D5A4E` | 主题色，主按钮背景、链接、选中态描边 |
| `--color-brand-hover` | `#245048` | 按钮悬停（加深） |
| `--color-brand-active` | `#1C3F38` | 按钮点击态 |
| `--color-brand-subtle` | `#E6EFED` | 主题色浅底（激活的选项卡背景、Tag 背景） |
| `--color-brand-muted` | `#A3C4BC` | 主题色淡化（进度条轨道背景） |

#### 1.1.5 强调色（Accent）

| Token | 值 | 用途 |
|-------|----|------|
| `--color-accent` | `#B94A3E` | 赤陶强调色，顶栏 Logo 点缀、重要提示标记 |
| `--color-accent-hover` | `#A33E33` | 悬停态 |
| `--color-accent-subtle` | `#F5E8E7` | 赤陶浅底，错误提示背景 |

#### 1.1.6 功能色（Functional）

功能色与暖白调性协调，避免高饱和原色：

| Token | 值 | 用途 |
|-------|----|------|
| `--color-success` | `#3A6B49` | 成功（深绿，比主题色更绿） |
| `--color-success-subtle` | `#E6F0E9` | 成功浅底 |
| `--color-warning` | `#8C6A1A` | 警告（暖黄棕，避免刺眼柠檬黄） |
| `--color-warning-subtle` | `#F5EDDA` | 警告浅底 |
| `--color-error` | `#B94A3E` | 错误（复用强调色） |
| `--color-error-subtle` | `#F5E8E7` | 错误浅底 |
| `--color-info` | `#3D5E8C` | 信息（偏暖的蓝灰，与暖白融合感强） |

#### 1.1.7 诊断分级色（Diagnostics）

映射 F-002 AC-005 / F-011 兼容性报告三级：

| Token | 值 | 含义 |
|-------|----|------|
| `--color-diag-safe` | `#3A6B49` | 绿色 — 安全，无微信过滤风险 |
| `--color-diag-warn` | `#8C6A1A` | 黄色 — 可能降级 |
| `--color-diag-error` | `#B94A3E` | 红色 — 会丢失 |

### 1.2 排版 Token

#### 1.2.1 字体栈

```css
/* 正文衬线体（编辑器内正文、内容预览正文） */
--font-serif: "LXGW WenKai", "Source Han Serif SC", "Noto Serif SC",
              Georgia, Charter, "Times New Roman", serif;

/* UI 无衬线体（按钮、标签、菜单、表单） */
--font-sans: "Inter", system-ui, -apple-system, "PingFang SC",
             "Source Han Sans SC", "Noto Sans SC", sans-serif;

/* 代码等宽体（源码编辑器、内联代码、代码块） */
--font-mono: "JetBrains Mono", "Fira Code", "Cascadia Code",
             "Source Code Pro", ui-monospace, "Courier New", monospace;
```

| `--font-size-base` | `15px` | 主正文、UI 基础文字 |

### 1.5 动效 Token

| Token | 值 | 用途 |
|-------|----|------|
| `--duration-instant` | `80ms` | 按钮色变、focus ring 出现 |
| `--duration-fast` | `150ms` | Hover 态切换、Toast 入场 |
| `--duration-base` | `250ms` | 下拉菜单展开、抽屉展开 |
| `--duration-slow` | `350ms` | Modal 出场、Command Palette 展开 |
| `--ease-standard` | `cubic-bezier(0.4, 0, 0.2, 1)` | 大多数 UI 过渡 |
| `--ease-decelerate` | `cubic-bezier(0, 0, 0.2, 1)` | 元素入场（从快到慢） |
| `--ease-accelerate` | `cubic-bezier(0.4, 0, 1, 1)` | 元素退场（从慢到快） |

## 4. 导航与路由

### 4.1 路由表

| 路由 | 页面 | 描述 |
|------|------|------|
| `/` | P-001 编辑器主页 | 默认打开最近一篇文档，无文档时展示新建引导 |

| `/themes` | P-003 主题市场 | 主题浏览与安装 |
| `/settings` | P-004 设置页 | 用户偏好、API 密钥、同步配置 |
| `/preview/:docId` | P-005 只读预览 | 移动端 / 分享预览（仅展示内容，无编辑能力） |

### 4.2 导航行为

- 从任意页面通过顶栏 Logo 或快捷键 `Alt+H` 返回编辑器主页
- 路由切换保留编辑器滚动位置（IndexedDB 持久化）
- 未保存更改时切换文档展示「离开确认」对话框
- Command Palette（Ctrl+K）在任意路由均可触发，视图操作按当前路由上下文执行

### 4.3 左侧面板 Tab 序列

来自 Q3 决策，Tab 顺序固定：

1. **主题**（ThemeSelector）— 默认展开
2. **组件**（BlockLibrary）— 可搜索的 Block/Mark/Variant 插入器
3. **文档**（DocumentList）— 多文档列表管理
